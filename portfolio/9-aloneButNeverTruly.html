This game was created for the Midlands Industry 2021 global game jam and I teamed up with a musician called Adam. Adam had the idea of having a ship flying in space, I wanted to add something meta so I discussed having the ship searching for itself and finding itself through opening up a second instance of the game.

This idea turned out very well in the end. We didn't need to design any levels because the game revolved around the single puzzle of realizing you could open the game twice, and I could copy in the cutscene manager to add context and push the player towards that solution.

Connecting the two game instances ended up being a very big task. Fortunately, when I discussed this with Adam, he had experience with Unity and art and was happy to create a few meteors and to code up some basic things like the player ship controller.

In order to connect the two game instances, I took advantage of Unity's PlayerPrefs saving system. To test this in the editor, I had to stop the application closing code from running in the editor, and then I had to create a tool that would allow me to either run that closing code or to reset all of the saved data manually. This was useful and it actually revealed a bug where if a game instance closed down without properly running the closing code (ie if the game crashed) it would leave the save data open without it being connected to a running instance. I had to check for this using a diagnostics check on the number of Unity instances running.

We brought on a writer, Izzy, near the end of the jam and she did a very good job of writing up the pieces of dialogue we needed very quickly. I had to request some edits from her first writeup to make it more clear to the player what they need to do, but her second writeup was spot on.

The game seems to have been received quite well, only three people worked on the game including me, and I did share it with 2-3 friends, but it seems to have gotten 19 likes on the global game jam website.
